## Resumen Técnico

Dashboard administrativo para sistema POS con gestión multi-sucursal. Arquitectura cliente-servidor con API REST, caché en Redis para reportes de alta frecuencia, y PostgreSQL como fuente de verdad.

## Notas de Laboratorio

<LabNotes>
  <LabNotes.Note date="2023-07-20" topic="Estrategia de caché">
    **Problema**: Reportes de ventas tomaban 3+ segundos con datos de 6 meses.

    **Decisión**: Redis con invalidación por evento (nueva venta → invalidate cache de reportes del día). Latencia reducida a 50ms.
  </LabNotes.Note>
</LabNotes>
