## Resumen Técnico

Kumoriya es un sistema distribuido diseñado para unificar múltiples fuentes de contenido en una experiencia coherente. La arquitectura se divide en tres capas: presentación (Flutter), lógica de negocio (Go/Fiber) y edge computing (Cloudflare Workers).

El sistema implementa una arquitectura **plugin-first** donde cada fuente de contenido es un plugin independiente, permitiendo extensibilidad sin modificar el core.

## Notas de Laboratorio

<LabNotes>
  <LabNotes.Note date="2024-03-15" topic="Decisión de arquitectura">
    **Pregunta**: ¿Centralizado o distribuido?

    **Exploración**:
    - Opción A: Monolito simple (rápido, limitado)
    - Opción B: Microservicios (complejo, escalable)
    - Opción C: Edge distribuido (novedoso, resiliente)

    **Decisión**: Opción C. Se adapta mejor a constraints de latencia y disponibilidad.
  </LabNotes.Note>

  <LabNotes.Note date="2024-05-20" topic="Sincronización offline">
    **Problema**: Los usuarios necesitan acceso sin conexión.

    **Exploración**:
    - Opción A: Cache simple con expiración
    - Opción B: Sincronización bidireccional con CRDTs
    - Opción C: Base de datos local-first con sync

    **Decisión**: Opción C con CRDTs para resolución de conflictos. Mayor complejidad inicial pero experiencia superior.
  </LabNotes.Note>

  <LabNotes.Note date="2024-08-10" topic="Autenticación sin contraseñas">
    **Contexto**: Usuarios target son tech-savvy, fricción de contraseñas es inaceptable.

    **Implementación**: WebAuthn + Passkeys como método primario. OAuth 2.0 como login social.
    Ed25519 para firma de JWT y tokens de sesión.
  </LabNotes.Note>
</LabNotes>
