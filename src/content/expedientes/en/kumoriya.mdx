## Technical Summary

Kumoriya is a distributed system designed to unify multiple content sources into a coherent experience. The architecture is split into three layers: presentation (Flutter), business logic (Go/Fiber), and edge computing (Cloudflare Workers).

The system implements a **plugin-first** architecture where each content source is an independent plugin, enabling extensibility without modifying the core.

## Lab Notes

<LabNotes>
  <LabNotes.Note date="2024-03-15" topic="Architecture decision">
    **Question**: Centralized or distributed?

    **Exploration**:
    - Option A: Simple monolith (fast, limited)
    - Option B: Microservices (complex, scalable)
    - Option C: Distributed edge (novel, resilient)

    **Decision**: Option C. Best fit for latency and availability constraints.
  </LabNotes.Note>

  <LabNotes.Note date="2024-05-20" topic="Offline synchronization">
    **Problem**: Users need offline access.

    **Exploration**:
    - Option A: Simple cache with expiration
    - Option B: Bidirectional sync with CRDTs
    - Option C: Local-first database with sync

    **Decision**: Option C with CRDTs for conflict resolution. Higher initial complexity but superior experience.
  </LabNotes.Note>

  <LabNotes.Note date="2024-08-10" topic="Passwordless authentication">
    **Context**: Target users are tech-savvy, password friction is unacceptable.

    **Implementation**: WebAuthn + Passkeys as primary method. Fallback to magic links.
    Zero-knowledge proof for identity verification.
  </LabNotes.Note>
</LabNotes>
